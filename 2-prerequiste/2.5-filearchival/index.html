File Compression and Archival

Viewing file sizes

  1. du with -sk shows the size of a file or directory in Kilobytes
$ du -sk test.img
  1. du with -sh shows the size of a file or directory in human readable format
$ du -sh test.img
  1. we can also use long list , ls -lh to print the size of the file.
$ ls -lh test.img

Archiving Files

  1. tar is used to group multiple files and directories into a single file. Hence it is specially used for archiving data.
$ tar -cf test.tar file1 file2 file3 
$ ls -ltr test.tar
  1. The tar command followed by -tf option followed by the tar filename is used to see the contents of the tarball.
$ tar -tf test.tar
  1. The tar command followed by -xf option followed by the tar filename is used to extract the contents from the tarball.
$ tar -xf test.tar
  1. tar command followed by -zcf option is used to compress the tarball to reduce its size.
$ tar -zcf test.tar

Compression and Uncompression

  1. Let us now look at the three popular ones
  • bzip2 (.bz2 extension)
  • gzip (.gz extension)
  • xz (.xz extension)
$ bzip2 test.img
$ gzip test1.img
$ xz test2.img
  1. The compressed files can be uncompressed by using the below commands
  • bunzip2
  • gunzip
  • unxz
$ bunzip2 test.img
$ gunzip test1.img
$ unxz test2.img
  1. Tools such as zcat , bzcat and xzcat allow the compressed files to be read without an uncompress 
$ zcat hostfile.txt.bz2
$ zcat hostfile.txt.gz
$ zcat hostfile.txt.xz

Searching for files and Patterns

  1. Run locate command followed by the filename you are searching as an argument. This should return all paths matching the pattern.
$ locate City.txt
  1. To manually update the DB, run the command updatedb and then run the locate command again
$ sudo updatedb
  1. find
$ find /home/michael -name City.txt

Grep

  1. To search for the word second from the sample.txt
$ grep second sample.txt
  1. To search for the word capital with case-insensitive use -i flag.
$ grep -i capital sample.txt
  1. To search for a pattern recursively.
$ grep -r "thrid Line" /home/michael
  1. To print the lines that don’t matches the pattern
$ grep -v "printed" sample.txt
  1. Match a pattern that form a whole word
$ grep -w exam examples.txt
  1. You can also combine multiple options together. For example, to reverse the search and print all lines of the same file that doesn’t match the whole word exam. Use grep -vw
$ grep -vw exam examples.txt
  1. To print the number of lines after and before matching a pattern. Use grep command with -A and -B flags respectively.
$ grep -A1 Arsenal premier-league-table.txt
$ grep -B1 4 premier-league-table.txt
  1. The -A and -B can be combined into one single search. 
$ grep -A1 -B1 Chelsea premier-league-table.txt

IO Redirection

In this section, we will take a look at IO Redirection.

  • IO Redirection
  • Standard Streams in Linux

There are three data streams created when you launch a linux commnad.

  • Standard Input (STDIN) - STDIN is the standard input stream which accepts text as an input.

  • Standard Output (STDOUT) - Text output is delivered as STDOUT or the standard out stream

  • Standard ERROR (STDERR) - Error messages of the command are sent through the standard ERROR stream (STDERR)

REDIRECT STDOUT

  1. To redirect STDOUT to a file instead of printing it on the screen.
$ echo $SHELL > shell.txt
  1. To append STDOUT to an exisiting file
$ echo $SHELL >> shell.txt

REDIRECT STDERR

  1. To redirect just the ERROR message we need to use 2 followed by forward arrow > symbol and then the name of the filename in which the errors are written.
$ cat missing_file 2> error.txt
  1. To append the STDERR to the exisiting file
$ cat missing_file 2>> error.txt
  1. If you want to execute and not print ERROR messages on the screen even if it generates a standard ERROR. You can redirect to /dev/null
$ cat missing_file 2> /dev/null

Command Line Pipes

  1. Command Line Pipes allow the linking of multiple commands.
$ grep Hello sample.txt | less

STDIN and STDOUT is the tee command.

  1. Instead of the redirect operator, we can use the command line pipe (|) followed by tee command.
$ echo $SHELL | tee shell.txt
  1. Use tee with -a option, to append instead of overwritting it
$ echo "This is the bash shell" | tee -a
